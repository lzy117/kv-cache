# A2 实验目标：KV 显存账本与模型选型

## 本阶段要解决的问题

A2 要回答一个非常基础但关键的问题：在一张 24GB GPU 上，不同模型大约能留下多大的 KV cache 池？

如果不知道 KV 池大小，后续 trace 的工作集规模就没有依据。trace 太小不会触发淘汰，trace 太大又会让所有策略都被极端压力冲垮。

## 需要完成的任务

1. 明确每 token KV cache 字节数的计算公式。
2. 核对候选模型的结构参数，包括层数、KV head 数、head_dim、精度字节数。
3. 估算 24GB GPU、`mem_fraction_static=0.85` 下的 KV 池大小。
4. 估算每个候选模型大约能缓存多少 token。
5. 给出主实验模型和调试模型选择。

## 候选模型

- Llama-2 7B：作为反例，验证 MHA 模型 KV 太大。
- Qwen2.5-7B-Instruct：主实验候选。
- Qwen2.5-1.5B：快速调试和扫参候选。

## 对整个实验的作用

A2 会决定后续 W1-W4 trace 的尺度。我们要把工作集设计成 KV 池的 2-5 倍左右，让淘汰真实发生但不过度极端。这个阶段也是后续 GPU 租用前控制成本的基础。
