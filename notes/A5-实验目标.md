# A5 实验目标：完整 2Q 策略设计与实现

## 本阶段要解决的问题

A5 要实现一个带状态的完整 2Q 淘汰策略，用来验证“幽灵队列能否抵抗扫描污染”。

上游当前的 `get_priority(node)` 是无状态接口，只能根据单个节点的字段打分。完整 2Q 需要记录节点经历过哪些队列、是否曾经被从试用区淘汰、是否应该在二次访问时晋升到主缓存区。因此 A5 必须扩展淘汰策略接口，给 RadixCache 增加事件钩子。

## 需要完成的任务

1. 撰写 `notes/04-2q-design.md`，明确完整 2Q 的状态机。
2. 给 vendor 版 `EvictionStrategy` 增加向后兼容的可选钩子。
3. 在 vendor 版 `RadixCache` 中调用这些钩子。
4. 实现 `TwoQStrategy`。
5. 在模拟器中加入 `2q` 策略。
6. 编写 CPU 单测，验证 2Q 的晋升、淘汰和扫描抵抗行为。
7. 在 A4 模拟器中补跑 2Q，更新 `results/sim_matrix.csv`。

## 2Q 的三个区域

- A1in：试用队列。新节点先进入这里，按 FIFO 淘汰。
- A1out：幽灵队列。只记录从 A1in 淘汰出去的 key 历史，不占 KV 池。
- Am：主缓存区。命中过 A1out 的节点晋升到这里，按 LRU 管理。

A5 是本项目最有原创性的部分。
