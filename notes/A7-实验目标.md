# A7 实验目标：租卡前 Gate 检查

## 本阶段要解决的问题

A7 是租 GPU 前的放行关口。目标不是继续扩展实验，而是确认 A1-A6 的理论、代码、脚本、结果和仓库状态已经足够稳定，避免租卡后才发现基础链路缺失。

本阶段要回答三个问题：

1. A1-A6 的关键产出是否齐全？
2. 2Q patch、replay/collect、模拟器和 mock 验证是否能在 WSL 中复现？
3. 租用 GPU 机器后，是否有明确的 setup 脚本和实验排期？

## 需要完成的任务

1. 补齐 A1 的 evict 流程图，使机制笔记满足 Gate 要求。
2. 导出可应用到 SGLang `v0.5.14` 的 2Q patch。
3. 编写 `scripts/setup_gpu.sh`，用于 GPU 机器初始化。
4. 编写主实验矩阵与预计时长表。
5. 运行 A7 Gate 检查，包括单测、模拟矩阵核对、mock replay/collect、脚本语法、密钥扫描、ignored 产物检查。
6. 撰写 `notes/A7-Gate检查报告.md` 和 `notes/A7-实验总结.md`。

## 本阶段预期产出

- `notes/A7-实验目标.md`

如果所有可本地验证项目通过，可以进入 B1/B2。

如果仍有外部依赖项目，例如 GitHub remote 尚未配置、真实 GPU 环境尚未安装，则必须在 Gate 报告中标为“待人工完成”，不能假装全绿。
