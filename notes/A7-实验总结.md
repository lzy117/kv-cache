## 关键发现

1. 上游 `server_args.py` 默认 `RADIX_EVICTION_POLICY_CHOICES` 不包含 `2q`。因此 patch 里不仅要改 `evict_policy.py`、`radix_cache.py`、`utils.py`，还必须把 `2q` 加入 CLI choices。
2. patch 文件必须是 ASCII/LF。PowerShell 生成的 UTF-8 BOM patch 会让 `git apply` 报错，因此最终用 WSL 重新生成。
3. 本地路径包含空格时，`git apply` 直接读取 patch 路径可能异常；`setup_gpu.sh` 会先复制 patch 到 `/tmp` 再应用。
4. W4 结果支持“2Q 抗 LRU/FIFO 扫描污染”，但不支持“2Q 超过 SLRU/LFU”。这个结论需要保留到后续报告中。
