# 简历项目描述

## 项目名称

SGLang KV Cache 淘汰策略实验与 2Q 策略实现

## 简历版本

基于 SGLang 0.5.14 实现并验证 KV Cache 淘汰策略优化实验，阅读 RadixCache 与 eviction policy 源码，新增 2Q 淘汰策略并接入真实服务端 CLI；构建 W1-W4 多类请求 trace，覆盖 few-shot 前缀复用、多轮对话、Zipf 热点租户与扫描污染场景；开发 replay/collect/scan-revisit 分析脚本，在 RTX 3090 + Qwen2.5-7B-Instruct 上采集 cached_tokens、hit rate、TTFT 等指标。实验发现 2Q 在扫描污染压力下略优于 LRU、与 SLRU 基本持平，而 LFU 在稳定高频热点场景下表现最佳，并据此总结了当前 2Q 实现与完整 2Q 机制之间的差距。

## 可拆分要点

- 深入分析 SGLang RadixCache、KV cache eviction policy 与 CLI 参数注册流程，实现 TwoQStrategy 并完成真实 GPU 服务接入。
- 构建可复现实验框架，支持 trace 生成、SGLang `/generate` 回放、cache flush、raw JSONL 记录与 CSV 汇总。
- 设计 W4 pressure workload，通过 hot -> scan -> hot 回访模式评估扫描污染下的热点保护能力。
- 在 Qwen2.5-7B-Instruct 上完成 LRU、SLRU、LFU、2Q 对比实验，并用 scan 后 32 条热点回访请求进行细粒度分析。

## 面试展开点

1. 为什么 KV cache 命中会影响 TTFT。
2. RadixCache 如何利用公共前缀复用 KV。
3. LRU、SLRU、LFU、2Q 的淘汰信号分别是什么。
4. 为什么当前 2Q 与 SLRU 结果接近。
5. 为什么 LFU 在稳定高频热点场景下反而更强。
6. 如果继续优化，如何实现严格 A1in/A1out/Am 配额和 ghost hit 统计。
