> 规则：本文件用于记录实验前预测。进入 GPU 实验后，不再修改本文件中的预测内容；如果预测错误，在后续结果分析中解释原因。

## 总体判断

RadixCache 的核心约束是“只淘汰叶子”。因此，策略差异不会只由 LRU/LFU/FIFO 本身决定，还会强烈受到树形态影响。

总预测：

- 共享前缀越多、内部节点越多，策略差异越小。
- 独立长会话越多、叶子层选择越多，近期性策略越有优势。
- 访问频率越倾斜，LFU/SLRU/2Q 越容易优于 FIFO。
- 一次性扫描流量越强，2Q 相对 LRU 的优势越明显。

## W1 few-shot 评测

### 负载特征

W1 模拟多个学科的 few-shot 评测。每个学科共享固定 5-shot 前缀，请求按学科交错发出。

树形态预期：

- 每个学科有一个较热的共享前缀；
- 共享前缀容易成为内部节点；
- 叶子主要是每个 held-out question 的短尾部。

### 策略预测

预测：**策略差异较小，LRU≈LFU≈SLRU，FIFO 不应显著崩坏**。

可证伪判断：

- 在中压下，主流策略之间命中率差距预计小于 5 个百分点。
- 如果 W1 中某个策略比 LRU 高出超过 10 个百分点，需要回头检查 trace 是否没有形成足够共享前缀，或模拟器是否把内部节点也当成可淘汰对象。

### 机制解释

W1 的公共前缀会被 RadixCache 的树结构自然保护。由于策略只能选择叶子，真正被策略比较的是短问题尾部，而不是大块 shared few-shot prefix。

## W2 多轮对话

### 负载特征

W2 模拟多个会话交错进行。每个会话的 prompt 随轮次增长，后续请求会重复包含同一会话的历史上下文。

树形态预期：

- 每个会话是一条逐渐加深的路径；
- 不同会话之间共享较少；
- 近期轮次更可能再次被访问。

### 策略预测

预测：**LRU 和 SLRU 优于 FIFO；LFU 不一定稳定占优**。

可证伪判断：

- 在高压下，LRU/SLRU 相对 FIFO 应至少高 5-8 个百分点。
- 如果 FIFO 与 LRU 持平，说明压力可能不足，或会话交错没有制造足够近期性。
- 如果 LFU 显著优于 LRU，需要检查 hit_count 是否被多轮历史重复插入放大。

### 机制解释

多轮对话的主要信号是近期性。刚访问过的会话更可能在下一轮继续出现，所以 LRU 应该能保留更有价值的叶子。FIFO 不看近期性，容易淘汰仍会继续使用的会话尾部。

## W3 Zipf 多租户 system prompt

### 负载特征

W3 模拟多个租户的 system prompt。请求按 Zipf 分布选择租户，少数热门租户承担大量请求。

树形态预期：

- 热门租户前缀反复出现；
- 长尾租户制造干扰；
- 访问频率比单纯近期性更重要。

### 策略预测

预测：**LFU/SLRU/2Q 应优于 FIFO，可能优于 LRU**。

可证伪判断：

- 在中压和高压下，LFU 或 SLRU 至少应有一个不低于 LRU。
- 如果 LRU 明显优于 LFU，说明 Zipf 热点的重访间隔可能太短，近期性已经足够表达热点。
- 如果 FIFO 接近最优，说明工作集压力可能不足，或者租户前缀被内部节点保护得过强。

### 机制解释

Zipf 场景中，热点租户会多次出现。LFU 用 `hit_count` 直接表达频率，SLRU 用 hit_count 阈值形成试用/保护段，2Q 则通过二次访问晋升过滤长尾噪声。

## W4 扫描污染

### 负载特征

W4 在 W3 的 Zipf 多租户基础上，周期性插入一次性长文档请求。扫描请求很长，但不会再次出现。

树形态预期：

- 多租户热点仍然存在；
- 扫描请求会创建很长的一次性叶子；
- 普通 LRU 容易被“新来的长叶子”污染。

### 策略预测

预测：**完整 2Q 应显著优于 LRU；LRU 可能被扫描流量洗掉热点；FIFO 也会较差**。

可证伪判断：

- 在高压下，2Q 相对 LRU 预计至少高 8-10 个百分点。
- 如果 2Q 无法超过 LRU，优先检查 A1in/A1out 配额、ghost 匹配逻辑和 split 状态继承。
- 如果 LRU 没有被扫描影响，说明扫描请求占池比例可能不足，或 RadixCache 叶子约束天然保护了热点前缀。

### 机制解释

2Q 的关键是 A1out 幽灵队列。一次性扫描请求即使进入 A1in，也不会因为“刚来过”就污染 Am。只有短时间内再次出现的内容才会晋升为主缓存热点。

## 对 A4 模拟器的要求

A4 模拟器跑完后，需要为每个负载输出：

- 各策略命中率；
- 命中率随池大小变化的曲线；
- 预测是否符合本文件；
- 如果不符合，解释是负载设计问题、模拟器实现问题，还是 A1 理解需要修正。

特别关注：

- W1 的策略差异是否真的小；
- W2 中 FIFO 是否明显差；
- W3 中 LFU/SLRU 是否体现频率优势；
- W4 中 2Q 是否能抵抗扫描污染。
